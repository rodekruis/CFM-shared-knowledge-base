# [TEST] Rapports et tableau de bord
 
Les rapports et le tableau de bord vous aident à :
 
- Comprendre **quels retour d'information les communautés partagent**
- Identifier **les tendances et les problèmes récurrents**
- Suivre **comment (bien/rapidement) les retour d'information sont traités**
- Soutenir **la prise de décision et les améliorations des programmes**

### Que pouvez-vous voir dans les rapports et le tableau de bord
 
#### 1. Volume des retour d'information
 
- Nombre total d'entrées de retour d'information
- Tendances au fil du temps (quotidiennes/hebdomadaires/mensuelles)
Vous pouvez utiliser cela pour :
 
- Voir si l'engagement augmente ou diminue
- Identifier les pics (par ex. après des distributions ou des campagnes de communication)

#### 2. Sujets / Catégories (Codage)
 
- Principaux secteurs qui reçoivent des retour d'information (par ex. transferts monétaires, abris, WASH)
- Sous-thèmes (basés sur les niveaux de codage)
Vous pouvez utiliser cela pour :
 
- Identifier les problèmes clés et les problèmes récurrents

#### 3. Localisation
 
- Retour d'information par région / zone
Vous pouvez utiliser cela pour :
 
- Identifier d'où proviennent les retour d'information
- Comparer les zones (par ex. une région a plus de plaintes)

#### 4. Canal de retour d'information
 
- Comment les retour d'information ont été collectés (téléphone, en face à face, etc.)
Vous pouvez utiliser cela pour :
 
- Comprendre quels canaux sont préférés par les communautés
- Améliorer la stratégie de sensibilisation

#### 5. Gestion des retour d'information (Statut et réponse)
 
Vous pouvez voir :
 
- Retour d'information Ouverts / En cours / Fermés
- Temps de réponse
- Suivi nécessaire
Vous pouvez utiliser cela pour :
 
- Suivre la performance dans la clôture de la boucle de retour d'information
- Identifier les retards

#### 6. Satisfaction
 
Vous pouvez utiliser cela pour :
 
- Comprendre si les réponses/le traitement des retour d'information sont **efficaces**
- Améliorer la communication et le suivi

### Comment interpréter les données
 
- **Recherchez des tendances, pas des cas isolés**
  - Une plainte = anecdotique
  - De nombreuses plaintes similaires = **véritable problème**
- **Faites attention à :**
  - Augmentation des plaintes
  - Problèmes répétés dans le même secteur
  - Retards dans la clôture des cas
  - Faibles scores de satisfaction
- **Combinez les points de données**
  - Exemple : de nombreuses plaintes + même localisation + même secteur → signal pour un ajustement du programme

### Comment utiliser les rapports
 
Lorsque vous êtes dans le tableau de bord, vous pouvez cliquer sur les différents rapports pour obtenir des données plus détaillées. Pour ce faire :
 
- Trouvez le rapport sur lequel vous souhaitez obtenir plus d'informations.
- Cliquez sur les 3 points en haut à droite du rapport,
- Cliquez sur 'View report'.
*[Captures d'écran : options du menu de la vignette du rapport]*
 
La page latérale suivante s'ouvrira alors.
 
*[Capture d'écran : page latérale de détail du rapport]*
 
Vous pouvez utiliser les rapports pour :
 
- Filtrer sur des informations spécifiques, par exemple :
  - Filtrer par secteur → comprendre les problèmes spécifiques au secteur
  - Filtrer par localisation → soutenir les équipes de branche
  - Filtrer par date → suivre les changements au fil du temps
- Aller aux formulaires de retour d'information connectés
- Approfondir des problèmes spécifiques
- Exporter des données pour partager des informations
*Remarque : le nom du projet doit être écrit correctement lors du filtrage sur les projets dans le rapport. La liste finale des titres de projet est disponible à partir du formulaire Kobo et peut être partagée par le Gestionnaire des retours d'information.*
 
### Transformer les enseignements en action
 
L'étape la plus importante lors de l'utilisation des rapports et du tableau de bord : demandez-vous toujours **"Que devrions-nous faire différemment sur la base de cela ?"**
 
Exemples :
 
- De nombreuses questions sur l'éligibilité → améliorer la communication sur l'éligibilité
- Plaintes concernant les retards → ajuster les processus
- Problème répété dans une zone → assurer un suivi local

### Bonnes pratiques
 
- Examiner le tableau de bord **régulièrement**
- Partager les enseignements avec les **équipes programme et sectorielles**
- Utiliser à la fois des **chiffres et des exemples (citations)**
- Garder les retour d'information liés à **la prise de décision**
Le système n'est pas seulement destiné au reporting. C'est un outil pour :
 
- Écouter les communautés
- Répondre à leurs besoins
- Améliorer les programmes
