# [TEST] Reports and Dashboard
 
The reports and dashboard help you:
 
- Understand **what feedback communities share**
- Identify **trends and recurring issues**
- Monitor **how (well/fast) feedback is being handled**
- Support **decision-making and programme improvements**

### What can you see in the Reports and Dashboard
 
#### 1. Volume of Feedback
 
- Total number of feedback entries
- Trends over time (daily/weekly/monthly)
You can use this to:
 
- See if engagement is increasing or decreasing
- Identify peaks (e.g. after distributions or communication campaigns)

#### 2. Topics / Categories (Coding)
 
- Main sectors that are receiving feedback (e.g. cash, shelter, WASH)
- Sub-themes (based on coding levels)
You can use this to:
 
- Identify key issues and recurring problems

#### 3. Location
 
- Feedback by region / area
You can use this to:
 
- Identify where feedback is coming from
- Compare areas (e.g. one region has more complaints)

#### 4. Feedback Channel
 
- How feedback was collected (phone, face-to-face, etc.)
You can use this to:
 
- Understand which channels are preferred by the communities
- Improve outreach strategy

#### 5. Feedback Management (Status & Response)
 
You can see:
 
- Open / In Progress / Closed feedback
- Response time
- Follow-up needed
You can use this to:
 
- Monitor performance on closing feedback loop
- Identify delays

#### 6. Satisfaction
 
You can use this to:
 
- Understand if responses/feedback handling are **effective**
- Improve communication and follow-up

### How to interpret the data
 
- **Look for patterns, not single cases**
  - One complaint = anecdotal
  - Many similar complaints = **real issue**
- **Pay attention to:**
  - Increase in complaints
  - Repeated issues in same sector
  - Delays in closing cases
  - Low satisfaction scores
- **Combine data points**
  - Example: many complaints + same location + same sector → signal for program adjustment

### How to use reports
 
When in the dashboard, you can click on the different reports to provide more detailed data. To do this:
 
- Find the report you want to get more information on.
- Click on the 3 dots on the top right of the report,
- Click on 'View report'.
*[Screenshots: report tile menu options]*
 
The following side page will then open.
 
*[Screenshot: report detail side page]*
 
You can use reports to:
 
- Filter on specific information, for example:
  - Filter by sector → understand sector-specific issues
  - Filter by location → support branch teams
  - Filter by date → monitor changes over time
- Go to the connected feedback forms
- Deep dive into specific issues
- Export data to share insights
*Note: the project name needs to be written in the correct way when filtering on projects in the report. The final list of project titles is available from the Kobo Form and can be shared by the Feedback Manager.*
 
### Turning insights into action
 
The most important step when using the reports and dashboard: always ask yourself **"What should we do differently based on this?"**
 
Examples:
 
- Many questions about eligibility → improve communication about eligibility
- Complaints about delays → adjust processes
- Repeated issue in one area → follow up locally

### Good practices
 
- Review the dashboard **regularly**
- Share insights with **program and sector teams**
- Use both **numbers and examples (quotes)**
- Keep feedback linked to **decision-making**
The system is not just for reporting. It is a tool to:
 
- Listen to communities
- Respond to their needs
- Improve programs
